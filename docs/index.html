The golden pattern

Four files. Every feature. Always.

This is the single most important rule about Spring Boot structure. Every feature you build follows the same four-file pattern — no exceptions.

💡
If you add Orders to your shop tomorrow, you create exactly the same four files: Order.java, OrderRepository.java, OrderService.java, OrderController.java. Same pattern, every time.

ANALOGY

Think of it like a company — the receptionist doesn't go into the warehouse, and the warehouse doesn't answer customer calls. Each person stays in their lane.

Quick reference

The structure at a glance

src/main/java/com/shop/ ├── model/ │ └── Product.java ← your data shape (fields, getters) ├── repository/ │ └── ProductRepository.java ← extends JpaRepository ├── service/ │ └── ProductService.java ← business rules live here ├── controller/ │ └── ProductController.java ← @RestController, @GetMapping… └── MyShopApplication.java ← the entry point (don't touch!)
✅
You're done with structure. From here on, every Spring Boot project you ever write will follow this exact pattern. It becomes muscle memory fast.
